Introduction

While on Linux terminal a developer has available a set of commands, some of them adds more speed on their work, and they are the following:

Note: If you are new to Linux terminal, you might find this video useful.

To open your terminal you can simply find its icon on your menu, or type the keyboard shortcut:

Ctrl + Alt + t

cd

Current Directory - used with ' ~ ' will lead us to the home directory.

cd ~

Typed with a directory name in place of '~' will open in terminal the given directory, useful to navigate in conjunction with the following 'ls' command.

cd Documments
ls

ls

Stands for List Storage, it will provide a list of what the directory has, in a list format.

It is useful for navigating in the directories with the previous "cd" command

ls
    The 'ls' has some vairations:
  • ls -a

    • Provides a list of everything, including the hidden files.

  • ls -l

    • Gives additional information about the files.

  • ls -la

    • Provides information about the hidden files.


Typing spaces on the terminal

This is not a command, but a way to type it. Using absolute path we will be able to get those files with spaces in their names.

Code example to open a folder with space in name: called new project; inside FCC folder located in Documents directory:

cd 'Documents/FCC/new-project'

Putting inside little quotes and detailing with the help of LS command we can now open folders and files with spaces in their names, remmembering to put one ' - ' where the space is located.


history

Will give the log of commands typed in until the last 1000 commands. Useful when you recently have forgotten some typed command and want to check it quickly.

history

man / whatis

The first stands for manual, and typed alone will show the manual of the terminal. Used in conjunction with other names after it, like a program name for example. It will provide the given manual, if available.

man firefox

The code above will bring the Mozilla firefox manual on the terminal

While the 'whatis' command will give us a short description of what the given command does.

whatis man

Resulting in:

man (1) - an interface to the on-line reference manuals

mkdir

mkdir for Make Directory, this command line provides us flexibility when operating only via terminal, like when adjusting files/folders before submitting to git-hub.

To create a new directory, type:

mkdir 'name of the desired directory'
    As we make new directories, there are some commands to undo them:
  • rm 'existing file name'

    • Will delete the file;

  • rm -r 'existing directory name'

    • Will delete the directory;

  • rmdir 'existing directory name'

    • Will delete the directories that have nothing inside;
